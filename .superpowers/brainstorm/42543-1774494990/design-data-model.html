Design Section 1: Data Model & Indexing Pipeline

How documents get chunked, indexed, and searched

New Prisma Model: DocumentChunk

prisma/schema.prisma
model DocumentChunk { id String @id @default(cuid()) documentId String document Document @relation(fields: [documentId], references: [id], onDelete: Cascade) chunkIndex Int // ordering within the document content String // raw text of this chunk (~500 tokens) heading String? // nearest heading above this chunk pageNumber Int? // for PDFs tokenCount Int // approx token count for context budgeting searchVector Unsupported("tsvector")? // PG full-text search // Future: embedding Float[] for pgvector createdAt DateTime @default(now()) @@index([documentId]) @@index([searchVector], type: Gin) // GIN index for fast FTS @@unique([documentId, chunkIndex]) }

Indexing Pipeline (on document upload)

📄
1. Extract Text

PDF → pdf-parse
DOCX → mammoth (already installed)
TXT/MD → raw read

✂️
2. Chunk

Split into ~500 token chunks
Respect heading boundaries
Overlap 50 tokens between chunks

🔍
3. Index

Generate tsvector per chunk
Store in DocumentChunk table
GIN index for fast search

✅
4. Mark Ready

Set Document.indexed = true
Store indexedAt timestamp
Ready for search

Search Flow (at query time)

Query: "What's our procedure for administering medication?"
Step 1: Convert query to tsquery → 'procedure' & 'administer' & 'medication'
Step 2: Search DocumentChunk with ts_rank() scoring, return top 8 chunks
Step 3: Group chunks by document, include heading + document title as context
Step 4: Stuff into Claude prompt as grounding material (~4000 tokens)
Step 5: Claude answers with citations: "According to your Medication Administration Procedure (Section 3)..."

Changes to Document Model

Add 3 fields to existing Document model:

  • indexed Boolean @default(false) — has this doc been chunked?
  • indexedAt DateTime? — when was it last indexed?
  • indexError String? — if extraction failed, store the reason
